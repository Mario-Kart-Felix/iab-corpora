When you connect two computers directly there is no Dhcp server so Windows assigns an APIPA (IP of form 169.254.x.y). If you connect a computer to a router (or to an ISP directly with a modem) you should get an IP assigned by a Dhcp server; if no Dhcp server is found the resulting APIPA indicates a problem. Windows seems to assume that you always want internet access so anytime it sees that an APIPA was assigned it warns about &quot;limited connectivity.&quot; In short, when connecting two computers directly ignore &quot;limited connectivity.&quot;<br/> <br/>
Here's a way to see if the two computers can communicate. If all the pings fail it is probably because of a firewall blocking, but could also be a cable or NIC problem. If the pings fail one direction only it is almost definitely a firewall.<br/> <br/>
To open a Command Window: Start - Run &#8211; cmd (command in 98SE) &#8211; OK<br/>
(if Run not on Vista&#8217;s or 7&#8217;s Start menu, just type cmd in the Search box)<br/> <br/>
To determine a computer's IP address: open a Command Window and type<br/>
ipconfig<br/> <br/>
Now for some troubleshooting:<br/> <br/>
1. On each PC open a Command window and type<br/>
ping IPofOther (where IPofOther is the IP address of the other PC)<br/> <br/>
Do you get Replies, or do you get packets lost?<br/> <br/>
If you got Replies,<br/> <br/>
2. Do the same ping except use the Computer Name of the other computer:<br/>
ping NameOfOtherPC<br/> <br/>

If one or both of the machine have gigabit adapters or auto-MDX ports, an Ethernet patch cable will do the job.<br/> <br/>
Configure the machines as follows.<br/> <br/>
PC #1:<br/>
IP address: 192.168.9.1<br/>
Subnet Address: 255.255.255.0<br/> <br/>
PC #2:<br/>
IP address: 192.168.9.2<br/>
Subnet Address: 255.255.255.0<br/> <br/>
Leave the Default Gateway and DNS Server addresses blank.<br/> <br/>
Make sure any firewalls are configured to put the 192.168.9.x subnet in the trusted zone.<br/> <br/>
Finally, you'll have to enable file/print sharing on the machines and actually share one or more folders or drives.<br/> <br/>
