	<p><p></p> an unshielded twisted pair (UTP) cable seems so simple, but it's caused countless hours of head scratching.</p> 

<p>The basic problem, as most administrators know, is that you need a "straight-through" cable to connect a regular hub or switch port to an end-node like a PC or a printer. This type of hub port is called MDI for "media dependent interface." But occasionally, administrators will want to connect two hubs or switches together. To accomplish this, you need an MDIX port on one of the devices, which simply switches the "pinouts" by crossing a pair. These ports are generally found on only smaller hubs, and rarely on switches. If you don't have an actual MDIX port you can simulate one by swapping the same pair in your cable, which we then call a "crossover" cable. So, the problem is that these crossover cables get tossed in with the regular cables because they often look identical, and when an unsuspecting user grabs one accidentally, it can take a while to troubleshoot them.</p> 

<p>Of course, over time, we've mostly learned the hard way to color code our crossover cables so we don't use them unintentionally, and for quite a while now, they've largely been replaced in the enterprise world by fiber, as administrators have taken to connecting their switches with high-speed Gigabit Ethernet.</p>

<p>But old technologies die hard, and <a href="http://en.wikipedia.org/wiki/Twisted_pair" target="_blank">twisted pair</a> is making a comeback. This is simple economics; electronics required for Gigabit Ethernet over copper UTP are much cheaper and easier to produce than the optics and lasers required for <a href="http://www.wise-intern.org/journal/2002/jasonfrederick.pdf" target="_blank">Gigabit Ethernet over fiber</a>.</p>

<p>But finally, after all these years, hardware manufacturers are implementing something that should have been commonplace on the earliest switches: the Auto-MDIX feature. <a href="http://en.wikipedia.org/wiki/Auto-MDIX" target="_blank">Auto-MDIX</a> lets you not only configure, in software, whether any given port is MDI or MDIX, but it also allows the switch to detect how the port should be configured.</p>

<p>There are a lot of implications to this strategy. One positive is that you can make all your cables straight-through now, and not worry about someone plugging in the wrong cable.</p> 

<p>A caveat, though, is that many network administrators have been using the lack of this feature as a method to control layer-2 bridge loops. That is, years ago, if someone plugged two switches together accidentally, the network would come crashing to a halt (unless Spanning Tree Protocol was properly configured), but only if they accidentally used a crossover cable, which was rare. With the auto-MDIX feature, which is enabled by default on some switches, and disabled by default on others, a user can far more easily create a problem because either type of cable can be used to connect two switches together. Unfortunately, this sort of problem used to be quite common.</p>

<p>But on a practical level, just about any switch that supports the Auto-MDIX feature also supports <a href="http://searchnetworking.techtarget.com/definition/spanning-tree-protocol" target="_blank">Spanning Tree Protocol</a>. Don't enable the former without the latter.</p>
